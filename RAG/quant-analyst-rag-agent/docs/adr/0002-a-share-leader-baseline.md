# ADR 0002：A 股主升浪识别采用数值行为、叙事证据与透明评分

- 状态：Accepted for Phase 0
- 日期：2026-07-13
- 决策版本：`cn-wave-rule-baseline-v0.1.0`

## 背景

“主升浪”是一个事后看图容易、事前定义困难的概念。只用阶段涨幅标注会产生未来信息泄漏，只用价格新高会混入大量假突破，只用题材文本又无法证明资金真正进入。

本项目要回答的研究问题是：

> 能否通过题材催化、基本面验证、量价突破、相对强度和筹码结构，建立一套可复现的 A 股主升浪龙头识别基线？

Phase 0 使用五段人工标注的历史正样本作为 research seed。它们不是训练完成的数据集，也不能用于宣称策略有效。负样本将在下一批标签中补充。

## 决策

系统将证据分成三个职责明确的部分：

1. SQL / dataframe 计算价格、成交额、换手、相对强度和筹码代理；
2. Retrieval 保存公告、投资者关系记录、新闻和研究笔记，并保留来源与可用时间；
3. 透明规则合成 `leader_score`，LLM 只归纳解释，不决定数字。

主升浪判断采用以下逻辑：

```text
资金是否真的进场：价格突破 + 成交额 + 相对强度 + 筹码代理
资金为什么持续进场：题材催化 + 基本面 / 预期验证
它是不是龙头：全市场和行业横截面相对强度
是否接近衰竭：天量滞涨 + 长上影 + 加速后回撤
```

## 标签语义

`data/labels/leader_cases.csv` 中的区间是人工研究标签，不是输入特征。流水线只在输出阶段把日期与标签连接，用于检查 baseline 在已知案例上的状态轨迹。

当前四种 `leader_type`：

- `institutional_trend`：产业趋势、订单或业绩验证较强；
- `narrative_momentum`：市场叙事强，但直接基本面验证较弱或被澄清；
- `price_hike_material`：产品涨价和供需弹性驱动，需警惕单一产品预期；
- 后续负样本将增加 `failed_breakout`、`theme_follower` 和 `narrative_disproved`。

标签不能被用于给当日 `theme_score` 或 `fundamental_score` 赋值，否则会形成标签泄漏。

## Point-in-time 规则

叙事表同时记录：

- `published_at`：来源标注的发布时间；
- `available_at`：系统最早允许使用这条证据的时间；
- `source_url`：证据出处；
- `evidence_scope`：是否可以作为当时信息使用。

特征流水线只做 backward as-of join。2026 年 6 月的报道不会被填回 2026 年 4 月的启动日，事后年报也不会被用于解释公告发布前的信号。

## 数值特征定义

### 价格与突破

- `return_Nd = close_t / close_(t-N) - 1`；
- `rolling_high_Nd` 使用截至当日的收盘价窗口；
- `distance_to_120d_high = close / rolling_high_120d - 1`；
- `new_high_count_20d` 统计最近 20 日触及 120 日收盘新高的次数。

使用收盘新高而非盘中最高价，是为了让信号在收盘后可复核。后续可把“突破前高”和“收盘确认”拆成两个特征。

### 成交额与换手

- `amount_ratio_20d = 当日成交额 / 前 20 日平均成交额`，均值不包含当日；
- `turnover_20d_sum` 与 `base_turnover_sum_60d` 使用小数单位，`1.0` 表示累计换手 100%；
- `amount_rank_market` 只有在当日证券数量至少达到 1,000 时才计算，否则为缺失。

最后一条防止把四只研究样本内部的排名伪装成全 A 股排名。

### 涨停与大阳线

- 主板默认 10%，创业板 / 科创板默认 20%，北交所默认 30%；
- `limit_up_count_10d` 使用对应板块阈值减 0.5 个百分点作为价格舍入容差；
- `large_up_count_10d` 统计单日收益大于 7% 的交易日。

Phase 0 未处理 ST 股票 5% 限制及历史制度变更；正式全市场版本必须从证券状态表按日期确定涨跌停制度。

### 筹码代理

仅有日频 OHLCV 时无法知道真实股东持仓成本。Phase 0 明确使用 proxy：

1. 用 `(high + low + close) / 3` 作为每日典型成交价格；
2. 用成交额作为权重；
3. 将最近 60 个交易日划分为 20 个价格区间；
4. `chip_concentration_60d` 为成交额最大的三个区间占比；
5. `avg_cost_distance` 为当前收盘价相对加权平均成本的距离；
6. `overhead_supply_ratio` 为典型价格高于当前收盘价的成交额占比。

这些字段只能称为筹码结构代理，不能描述为真实持仓分布。

### 衰竭代理

`high_volume_stall_flag` 第一版定义为：

```text
amount_ratio_20d >= 2
and return_1d <= 3%
and upper_shadow_ratio >= 3%
```

它只是一条风险标记，不等价于卖出信号。

## 评分

评分规则与用户给出的最小 baseline 一致：

- 题材：强题材 +2，直接相关 +1，叙事冲突 -1；
- 基本面：验证分不低于 2 时 +2；
- 突破：接近 120 日高点 +2，接近 250 日高点 +1；
- 量能：成交额放大 +2，全市场成交额 Top 100 / Top 50 各 +1；
- 相对强度：跑赢市场 +1，跑赢行业 +1，全市场 Top 5% +2；
- 筹码：60 日累计换手充分 +1，上方供给代理较低 +1；
- 衰竭：天量滞涨 -2，放量长上影 -1。

其中总分表达式是“各正向组件相加，再加负的衰竭组件”。不会把基本面、突破等正向分数相减。

阶段阈值：

```text
score >= 12  -> confirmed_main_uptrend
score >= 9   -> breakout_candidate
score >= 6   -> theme_strength_only
score < 6    -> normal
```

若 15 条规则中可计算比例低于 65%，输出 `insufficient_evidence`。如果加速上涨后出现天量滞涨，则优先输出 `exhaustion_risk`。

## 缺失值决策

缺失不是 0。

- 没有公告 / 新闻证据时，不把题材分默认为 0；
- 没有完整市场截面时，不计算全市场成交额和相对强度排名；
- 没有行业 / 题材历史成分时，不计算相应 RS；
- 每一行输出 `score_coverage` 和 `missing_components`。

这样可以区分“已验证但不满足规则”和“目前没有证据判断”。

## 数据调整与限制

Phase 0 使用 AkShare 提供的当前前复权日线，并记录实际 provider 与 `adjustment`。采集优先使用新浪日线，东方财富日线作为分年回退源；同一只股票的一次构建不会混用两种复权序列。它适合建立研究原型，但不是严格的 point-in-time 复权数据库。正式回测需保存原始价格、公司行动和按当时可知信息构造的复权因子。

当前仍缺少：

- 全 A 股历史成交额截面；
- 按日期生效的行业和题材成分；
- ST、停牌、上市 / 退市和涨跌停状态；
- 负样本及未走出股票的对照组；
- 文本证据覆盖率和多来源冲突解决机制。

因此 Phase 0 只验证数据契约、特征定义、时间语义和评分可运行，不进行策略胜率或收益宣称。

## 被否决的方案

### 只按未来区间涨幅训练分类器

容易得到漂亮结果，但会把事后结果当成事前特征，也无法解释假突破和衰竭。

### 让 LLM 直接给股票打分

数值不可复现、证据时间不可控、同一问题可能得到不同答案，不适合作为 baseline。

### 在四只样本中计算“全市场排名”

语义错误，会夸大量能和相对强度。Phase 0 选择保留缺失并报告覆盖率。

## 后果

优点：规则可复核、缺失透明、叙事不会穿越、后续可以逐个替换更强特征。

代价：Phase 0 分数不完整，部分已知龙头可能因行业 RS 和全市场排名缺失而得分偏低。这是数据覆盖不足的真实反映，不通过归一化补分掩盖。
